**前端 Agent 开发岗位（聚焦 AI 前端工程化方向）**  
*注：综合阿里、小红书、快手、字节、腾讯等大厂面经，聚焦“前端+AI Agent”交叉能力，非纯后端或算法岗*

- **技术类问题**：  
  1. 如何设计一个 AI Agent 工作流可视化编排画布（如 LangFlow）？技术选型（React + Canvas / X6 / Flowise）、核心难点（动态节点连接、状态同步、跨框架兼容）？  
  2. 实现流式“打字机”输出：逐字渲染 vs 分片缓冲？如何优化长 Markdown/LaTeX 实时渲染性能？如何处理中断、重连与断点续传？  
  3. 前端 RAG 流程参与点：文档预览锚点跳转、引用高亮、检索结果轻量重排、上下文截断策略（滑动窗口/摘要压缩）。  
  4. 多模态输入支持：设计可复用 Agent 输入框（图片/文件上传、元数据提取、格式校验、分块上传）。  
  5. 端侧 AI（WebLLM）关键技术点：WASM 加载、内存限制、模型量化、推理延迟监控、降级兜底（fallback 到服务端）。  
  6. AI 安全与鲁棒性：Prompt 注入防御（前端输入清洗+服务端校验）、幻觉 UI 缓解（置信度提示、编辑态回退、人工审核入口）。  
  7. Agent 状态管理：复杂多轮对话中，如何设计前端记忆机制（本地存储+服务端同步）、Token 上下文管理、消息裁剪策略？  

- **工程与架构类问题**：  
  1. 从零设计企业级 AI Agent 平台前端架构：分层设计（协议层 MCP/Skill、编排层 LangGraph.js、UI 层可插拔组件）、微前端集成方案。  
  2. 前端质量监控体系：AI 应用特有指标（流式响应耗时、幻觉率埋点、Tool 调用成功率、SSE 断连率）。  
  3. 性能优化专项：长列表（AI 动态高度项）、大文件解析（前端 FFmpeg.wasm）、音频可视化（WebAudio + FFT）、FCP/LCP 主动治理。  

- **行为与情景类问题**：  
  1. 如何用非技术语言向产品经理解释 Agent 工作流、Function Calling 或 RAG 的价值与边界？  
  2. 当 AI 生成代码存在幻觉或逻辑错误，你的前端 Review 流程和协作机制是什么？  
  3. 在快速迭代的 AI 技术中，如何平衡自研组件（如 Chat 组件、Tool 调用 SDK）与成熟库（LangChain.js、Vercel AI SDK）？  
  4. 面对“前端是否会被 AI 替代”的质疑，你如何定位自身不可替代性？（强调工程闭环、体验设计、安全兜底、跨端协同能力）  

**前端 Agent 开发面试流程**  
- 初试（45–60min）：基础+项目深挖——聚焦简历中 AI 相关项目（Agent 架构、流式交互、RAG 实践），考察技术真实性与原理理解；常含手撕（防抖、TaskRunner、流式渲染逻辑）。  
- 复试（60–90min）：系统设计+场景推演——要求现场设计 Agent 可视化平台、质量监控方案或端侧 AI 方案；高频追问容错、降级、监控、灰度等工程细节。  
- 终面（30–45min）：技术视野+角色认知——探讨 AI 对前端角色的重塑、个人技术护城河、长期学习路径及业务落地思考。
